# ThermalForge Agent 前端设计规范

## 1. 适用范围

本文档定义 ThermalForge 当前 Agent 前端的视觉语言、信息架构和交互原则。

- 唯一产品形态是“模型优先的单页 Agent 体验”。
- 七步工作台、顶部步骤导航、永久侧栏、指标卡网格不属于当前设计。
- 视觉参考基线为外部 `shiligongfang/frontend` 项目；取样文件包括 `index.html`、
  `App.tsx`、`components/AIChat.tsx`、`Dashboard.tsx` 和 `SafetyOps.tsx`。该项目主要
  通过 Tailwind Play CDN 配置和组件 `className` 表达样式，并非传统独立 CSS。
- 参考项目只提供视觉语法，不形成运行时依赖，也不复制其仪表盘信息架构。
- 视觉实现以 `thermalforge-studio/src/index.css` 和
  `thermalforge-studio/src/styles/agent.css` 为事实源。
- 页面结构以 `thermalforge-studio/src/agent/AgentExperience.tsx` 为事实源。

迁移规则：

| 类型 | 处理 |
|---|---|
| 原值照搬 | `#050505` 页面底色、`#121212` 卡面、白色 5%/10% 边界、`#888888` 弱文字、`#ff5500` 品牌橙 |
| 配方照搬 | 双角环境光的尺寸/透明度/模糊值、橙色主按钮渐变和品牌阴影 |
| Agent 适配 | 底部上扬橙光、输入区黑色渐隐和表面微渐变；不复制卡片网格、永久侧栏或顶部胶囊导航 |
| 明确保留 | IBM Plex 字体、模型优先结构、阶段状态机、工程语气和当前响应式断点 |

## 2. 设计定位

风格关键词：**深黑工业、微渐变、克制发光、模型优先、工程可信**。

核心画面是一件悬浮在近黑空间中的工程结构。视觉语言继承“视锂工坊”的深黑基底、
橙色低透明度光场和细白边界；Agent 围绕模型按里程碑推进任务，界面本身退居背景，
不与模型争夺注意力。

设计不是：

- 传统后台仪表盘；
- 多步骤表单工作台；
- 左右分栏聊天软件；
- 高饱和赛博朋克；
- 大面积玻璃拟态或装饰性霓虹。

## 3. 体验原则

### 3.1 Model first

3D 模型始终是首屏面积最大、层级最高的内容。输入、进度和结果说明围绕模型展开，
不能把模型压缩成普通卡片或预览缩略图。

### 3.2 One screen, one primary action

首屏只突出当前阶段最重要的动作：提交、停止、补充信息或新建设计。次要操作使用
低对比度按钮，不与热橙色主操作竞争。

### 3.3 Progressive disclosure

- 主界面只展示最近对话和当前进度。
- 完整过程进入底部历史抽屉。
- 部件参数只在选中模型部件后显示。
- 错误、上传状态和补充问题在相关上下文中出现。

### 3.4 Honest engineering

状态必须对应真实任务阶段。加载、重连、失败、取消和完成使用不同文案与视觉状态，
不能把处理中包装成已完成，也不能隐藏可恢复错误。

## 4. 页面骨架

页面固定为四层：

1. **顶部栏**：品牌、当前任务、会话状态和“新建设计”。
2. **中央模型舞台**：3D 模型、阶段读数、模型控制和交互提示。
3. **底部对话层**：最近消息、1px 进度线、文件与文本输入。
4. **按需浮层**：完整过程抽屉、工程依据、脱敏后端输出、部件详情、加载或错误提示。

桌面端顶部栏高度为 `56px`，主体占据剩余视口。底部对话层最大宽度为 `760px`，
并通过渐变融入背景，不形成横贯页面的实体面板。

## 5. 色彩系统

当前 CSS 变量如下：

| Token | 值 | 用途 |
|---|---:|---|
| `--tf-bg` | `#050505` | 直接继承参考项目的页面底色 |
| `--tf-bg-elevated` | `#111111` | 对话主体、抽屉深层背景 |
| `--tf-surface` | `#121212` | 直接继承参考项目的基础卡面 |
| `--tf-surface-strong` | `#1a1a1a` | 输入器与更强控件表面 |
| `--tf-line` | `rgba(255, 255, 255, 0.05)` | 极弱结构边界 |
| `--tf-line-strong` | `rgba(255, 255, 255, 0.10)` | 输入器、浮层和可交互边界 |
| `--tf-text` | `#ffffff` | 主要文字 |
| `--tf-text-soft` | `#d1d5db` | 正文与次级标题 |
| `--tf-text-muted` | `#888888` | 标签、说明、非活动状态 |
| `--tf-orange` | `#ff5500` | 直接继承的品牌橙 |
| `--tf-orange-strong` | `#ea580c` | 主按钮渐变起点、底部环境光 |
| `--tf-orange-deep` | `#7c2d12` | 低透明度深橙环境层 |
| `--tf-orange-soft` | `#fb923c` | Agent 标签与柔和强调 |
| `--tf-success` | `#22c55e` | 完成、上传成功 |
| `--tf-danger` | `#f87171` | 错误、失败、取消 |

使用约束：

- 页面主体保持黑、灰和暖白，热橙只用于主操作与当前状态。
- 同一语义必须引用统一的 `--tf-*` 色阶，禁止组件各自发明近似十六进制色值。
- 成功绿和危险红只表达状态，不作为装饰色。
- 状态不能只靠颜色表达，必须同时提供文字或图标。
- 禁止新增无语义的高饱和辅助色。

## 6. 字体与排版

- 界面与正文：`IBM Plex Sans Variable`，回退到 `Segoe UI`。
- 技术参数、阶段编号、微型标签：`IBM Plex Mono`，回退到
  `Cascadia Mono` 和系统等宽字体。
- 字体在本地依赖中托管，不等待第三方字体服务。
- 参考项目使用 `Inter`，但此项不照搬；IBM Plex 是 ThermalForge 的工程识别资产。

排版层级：

- 品牌标题约 `16px`，移动端 `14px`。
- 输入正文 `14px`，移动端保持 `16px`，避免软键盘自动缩放。
- 对话正文 `12–13px`，行高约 `1.5–1.6`。
- 技术标签 `8–10px`，使用等宽字体和较宽字距。
- 大标题数量要少；通过留白、字重和颜色建立层级，不依赖巨大字号。

## 7. 空间、边界与表面

- 使用紧凑的 `4px / 8px` 节奏，常用间距为 `6px、8px、12px、16px、24px`。
- 边界统一为白色 5% 或 10% 的 `1px` 线条。
- 常规控件圆角为 `7–11px`；输入器和详情面板为 `15px`。
- 用户消息使用 `11px 11px 3px 11px` 的非对称圆角。
- 主内容不包裹成卡片；实体表面只用于输入器、浮层、控制按钮和用户消息。
- 阴影只用于表达层级；品牌发光必须低透明、单层且紧邻当前操作。

背景只保留三层气氛效果：

1. 左下 `600px`、`blur(150px)`、`rgba(234, 88, 12, 0.10)` 的环境光；
2. 右上 `400px`、`blur(100px)`、`rgba(124, 45, 18, 0.10)` 的深橙环境光；
3. 模型后方与输入区各自的局部微渐变。

不再使用全屏噪点纹理。背景质感由低透明度渐变、纯黑留白和细边界建立。

### 7.1 微渐变配方

微渐变是本设计的关键，不得退化为纯色面板或大面积高亮：

| 场景 | 配方 |
|---|---|
| 全局环境光 | 照搬参考项目的 `600/150/10%` 左下橙光与 `400/100/10%` 右上深橙光 |
| 模型气氛 | `#ff5500` 18% 的中心径向渐变叠加 `#7c2d12` 8% 的偏移渐变 |
| 对话层渐隐 | 上方透明，经过 `#111111` 72%，落到 `#050505` 98%，再叠加底部橙色椭圆光 |
| 主操作 | 水平 `#ea580c → #ff5500`，阴影 `0 4px 20px rgba(249,115,22,0.40)` |
| 输入器 | `145deg` 的 `#1a1a1a → #111111`，只制造轻微材质差，不形成亮色渐变 |
| 浮层 | 按层级使用 `#141414/#161616 → #111111`；全宽历史抽屉使用 `#121212 → #050505` |
| 用户消息 | `#ea580c` 14% 到 `#7c2d12` 10%，配合品牌橙 20% 边界 |
| 上下文强调 | 深橙 18% 到 2% 的水平渐隐，用于标签和局部说明，不用于整页背景 |

同一视区最多出现一个主品牌渐变和两个低透明度环境渐变。禁止多层霓虹、彩虹渐变、
高饱和大色块和无语义持续发光。微渐变使用静态背景和伪元素实现，不在实时 WebGL
画布上叠加 `backdrop-filter`。

## 8. 核心组件规则

### 8.1 顶部栏

- 左侧为六边形品牌标记和 ThermalForge 名称。
- 中间仅在宽屏展示当前任务标识。
- 右侧展示带文字的会话状态；存在任务时才显示“新建设计”。
- 顶部栏保持单行、低高度，不承载导航菜单。

### 8.2 模型舞台

- 模型居中并占据绝大部分可用空间。
- 阶段编号、阶段名和百分比置于顶部中央。
- 后端提供多个模型版本时，在左上资产标签下显示整体/分件版本切换；当前版本使用热橙微渐变，
  但不抢占主操作层级。
- 模型控制靠右，桌面端按钮为 `50px`；自动旋转、线框、爆炸、重置和资产信息均提供
  可聚焦按钮。窄屏改为双列，低高度横屏进一步压缩并隐藏按钮文字。
- 爆炸只在清单提供节点绑定与偏移量时启用；不得用整体模型缩放伪装成分件爆炸。
- 部件详情是临时浮层，不做永久属性侧栏。
- 加载、WebGL 不可用和资产错误必须在舞台内给出明确反馈。

### 8.3 对话与输入

- 默认只显示最近消息，完整记录通过“完整过程”打开。
- Agent 消息使用无气泡排版；用户消息使用单层实体气泡。
- 输入器是底部最强表面，使用 `#1a1a1a → #111111` 微渐变，包含文件、文本和唯一主操作。
- 后端要求补充工程信息时，在输入器顶部展示深橙线性淡出的提示条；问题和回答输入
  保持独立，不能覆盖原始设计目标。
- 进度使用输入器上方的 `1px` 热橙线，不增加大型进度卡。
- 文件状态区分待上传、上传中、成功和失败。
- 运行阶段超过 30 秒没有收到新状态或事件时，在输入器上方显示克制的恢复提示，
  提供“重新检查”和“停止任务”，且保留已上传文件与任务上下文。

### 8.4 抽屉与详情

- 完整过程抽屉从底部进入，桌面端高度不超过 `620px`。
- 抽屉使用“对话 / 设计依据 / 后端输出”三视图；后端输出必须先递归清除密钥、令牌等
  敏感字段再显示。
- 桌面对话区可分为阶段轨道与消息；移动端阶段轨道改为横向滚动。
- 设计依据按选定方案、温度指标、传热路径、材料、风险和未验证项排列，不做仪表盘卡片堆叠。
- 部件详情桌面端位于模型右侧，窄屏改为底部浮层。
- 浮层关闭按钮、标题、指标和工程说明保持固定信息顺序。

## 9. 动效

| Token / 场景 | 时长 |
|---|---:|
| `--motion-fast` | `180ms` |
| `--motion-base` | `260ms` |
| 进度线更新 | `420ms` |
| 模型加载旋转 | `900ms` |

动效使用 `cubic-bezier(0.22, 1, 0.36, 1)` 的快速收敛曲线。消息轻微上移进入，
抽屉从底部进入，详情从右侧进入。持续动画仅用于当前运行状态或加载反馈；禁止无任务
语义的循环装饰动画。系统启用“减少动态效果”时，动画和过渡统一缩短至 `0.01ms`
并只执行一次。

## 10. 响应式

### `> 900px`

- 完整三列顶部栏；
- 详情面板在模型右侧；
- 历史抽屉使用阶段轨道 + 消息双列布局。

### `641px–900px`

- 隐藏顶部居中的任务标识；
- 详情面板转为底部浮层；
- 模型控制靠近右侧安全边距。

### `≤ 640px`

- 顶部栏降至 `52px`，隐藏品牌副标题和“新建设计”文字。
- 对话层宽度为视口减 `20px`。
- 最近消息最多显示三行。
- 历史抽屉高度为 `62vh`，阶段轨道横向滚动。
- 输入字号保持 `16px`，不允许横向页面滚动。

### 低高度横屏

当高度不超过 `680px` 且为横屏时，只显示一条最近消息，并隐藏模型交互提示，
优先保证模型与输入器可用。

## 11. 无障碍

- 保留“跳到主要内容”链接。
- 键盘焦点使用 `2px` 的 `--tf-orange-soft` 外轮廓，不能只靠颜色变化。
- 进度、状态和动态消息使用对应的 ARIA 语义。
- 图标按钮必须有可访问名称。
- 所有关键操作必须支持键盘完成。
- 页面最低支持 `320px` 宽度，并处理底部安全区。

## 12. 文案语气

- 使用简短、具体、可行动的中文。
- Agent 按工程里程碑汇报，不连续输出无意义过程日志。
- 错误文案说明“发生了什么”和“下一步能做什么”。
- 技术标签可以使用英文大写与等宽字体；用户操作保持自然中文。
- 不使用夸大、拟人化过度或无法验证的完成承诺。

## 13. 维护约束

新增或修改界面时必须检查：

- 是否仍以模型为首屏主角；
- 是否只保留一个主操作；
- 是否复用了现有颜色、字体、圆角和动效变量；
- 是否使用了正确的微渐变配方，而不是临时手写近似色；
- 是否避免重新引入步骤导航、永久侧栏和卡片网格；
- 是否覆盖加载、空态、错误、取消、恢复和窄屏；
- 是否保持键盘、焦点和 ARIA 语义；
- 是否保持静态 CSS 构建，未引入 Tailwind Play CDN 或其他运行时样式编译器；
- 是否同步更新本文档与实际 CSS。
