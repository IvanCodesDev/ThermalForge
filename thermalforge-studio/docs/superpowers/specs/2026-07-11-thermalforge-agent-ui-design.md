# ThermalForge 极简 Agent UI 重构设计

> **历史方案，已被当前实现取代。** 旧七步页面及其本地分析子图已移除；
> 当前视觉规范以仓库根目录 `docs/design.md` 和现行 Agent 源码为准。

状态：待产品确认  
日期：2026-07-11  
目标版本：Phase 0 交互骨架

## 1. 改造目标

将现有“7 个页面 + 顶部步骤导航 + 大量卡片表单”的工作流，重构为一个以 3D 模型为视觉中心的极简 Agent 页面。

用户只需要完成一件事：

> 上传硬件工程文档并描述设计目标，然后由 Agent 连续完成理解、方案生成、热设计优化、多视图生成和 3D 建模。

页面不是仪表盘、工作台或传统左右分栏聊天软件。它应更像一个安静的“3D 生成对话”：

- 中央始终保留最大的 3D 展示区域。
- 最近两条对话和输入集中在底部，不预先铺开复杂表单。
- Agent 只在关键节点播报结论，更早的消息按需展开。
- 参数、过程和技术细节按需展开。
- 生成完成后，用户直接操作模型，而不是跳转到结果页。

## 2. 本轮已确认的产品决策

1. 使用极简 Agent 交互，不延续当前工作台式布局。
2. 3D 模型固定为页面主角，位于视觉中心。
3. 底部保留极简对话区，默认只显示最近两条消息与输入框。
4. 采用“预建关节基座 + AI 生成可拆卸热增强外壳”的模型结构。
5. 点击整体模型后进入爆炸分解；点击热增强外壳后展示设计说明。
6. 3D 生成服务使用 Hyper3D Rodin。
7. 先实现可演示的 Phase 0 骨架，再逐段接入真实 LLM、图像生成、Hyper3D 和 Hermes Agent。
8. 现有热分析引擎和散热方案知识继续复用，不重新制造一套重复逻辑。

## 3. 明确不做的形态

以下模式在新界面中禁止出现：

- 永久在线的左侧聊天栏 + 右侧属性栏。
- 顶部 5～7 个步骤标签。
- 多列数据卡片、统计卡、状态卡。
- 一屏展示全部工程参数。
- 同时出现多个主要按钮。
- 为填满页面而添加的玻璃卡片、发光装饰和假进度条。
- 将完整对话历史长期占据主视区，或让对话区把 3D 模型挤到角落。
- 通过分页模拟流程推进。

## 4. 核心体验原则

### 4.1 Model first

3D 模型占据桌面端约 65%～75% 的有效视觉面积，并保持在视口水平中心与上半部视觉中心。底部对话层不得把模型压缩成普通预览窗。没有模型时，该区域显示克制的关节轮廓或生成中的几何预览，不使用大型空状态卡片。

### 4.2 One screen, one primary action

所有阶段都在同一个页面完成。任一时刻只强调一个主要动作：

- 初始：开始生成
- 失败：重试当前阶段
- 完成：查看设计
- 检视：收起说明

### 4.3 Progressive disclosure

默认只展示用户当前需要知道的内容。工程参数、解析结果、提示词、多视图和热分析数据通过“查看依据”或“设计详情”按需展开。

### 4.4 Agent speaks in milestones

Agent 不逐条展示内部思考，也不刷屏。只展示可验证的阶段结论：

- 已读取哪些文件
- 识别出的硬件对象和约束
- 选择了什么散热路线
- 当前正在生成什么
- 哪一步失败以及如何恢复
- 最终方案的关键指标和风险

主界面只保留最近两条消息。用户需要追溯上下文时，再展开完整历史。

### 4.5 Honest engineering

生成式模型无法保证工程方案和三视图“绝对正确”。产品必须把正确性建立在质量门禁上，而不是依赖一句 Prompt：

- 文档字段完整性校验
- 尺寸与安装包络校验
- 多视图几何一致性校验
- 热源、材料和散热路线规则校验
- 3D 模型尺度与朝向校验
- 最终方案明确标注“工程建议，生产前需复核”

## 5. 单页信息结构

### 5.1 顶部极简栏

高度控制在 56px 左右，仅包含：

- ThermalForge 标识
- 当前项目名或“未命名设计”
- 一个轻量状态点：空闲、生成中、已完成、失败
- 右侧溢出菜单：新建设计、打开历史、导出

不展示步骤导航、用户头像、保存时间胶囊或多组操作按钮。

### 5.2 中央 3D 舞台

中央舞台不是卡片，不使用明显边框。它直接融入页面背景，包含：

- 预建关节基座
- AI 热增强外壳
- 环境光、接触阴影和克制的热区颜色
- 旋转、缩放、重置视角三个基础控制
- “整体 / 爆炸”切换
- 当前选中部件的轻量描边

桌面端纵向构图固定为：

- 顶部极简栏约 48～56px。
- 3D 舞台使用剩余空间，并至少保留 55vh 的可视高度。
- 最近两条对话覆盖在舞台底部的渐隐区域，不另切一块大面板。
- 输入框固定在页面底部安全区内。

初始状态下，舞台中央显示低对比度关节轮廓和一句引导：

> 上传工程资料，生成可制造的热增强结构

生成过程中，舞台本身承担进度表达：

- 文档理解：关节轮廓逐渐清晰
- 方案生成：热路径从热点向外生长
- 多视图生成：模型周围短暂出现四向参考框
- 3D 建模：低模逐步替换为完整模型

### 5.3 底部极简对话区

对话区悬浮在页面底部中央，最大宽度约 760px。它由“最近两条消息 + 输入框”组成，不做成聊天侧栏，也不占用独立整列。

消息展示规则：

- 默认只渲染最近两条消息。
- Agent 消息左对齐，使用角色标签和纯文本，不套大型气泡。
- 用户消息右对齐，使用紧凑的单层深色气泡。
- 单条默认最多显示四行，超出内容提供“展开”。
- 新消息进入时旧消息向上淡出，不引起 3D 舞台尺寸跳变。
- 对话区顶部提供低强调的“查看全部”入口。

输入框默认内容：

- 文档上传按钮
- 一行到三行的设计描述输入
- 单个“开始生成”按钮

支持文件：

- PDF
- DOCX
- TXT / Markdown
- 常见工程图片

Phase 0 只读取文件名称、类型和大小，并使用 Mock 数据推进。真实文档解析在 Phase 1 接入。

提交后，输入框保持可见但进入任务态，主要动作变为“停止”；最近两条消息用于展示：

- Agent 的当前阶段说明
- 用户最近一次确认或补充

阶段名称与进度采用一行低强调状态文字，放在输入框上方，不增加新的进度卡片。

### 5.4 完整对话与过程抽屉

抽屉默认关闭，通过“查看全部”展开，最大高度不超过视口的 42%。展开时 3D 模型仍保持可见，不切换页面。

抽屉包含：

- 完整对话历史
- 阶段时间线
- 每阶段最终输出摘要
- 失败原因与重试入口

抽屉不展示模型内部推理或原始工具日志。关闭抽屉后立即恢复最近两条消息模式。

### 5.5 部件说明浮层

模型完成后：

1. 用户点击整体模型。
2. 基座、连接层和热增强外壳沿关节轴向分散。
3. 用户点击热增强外壳。
4. 页面右下或模型旁出现一个单独的说明浮层。

说明浮层只展示：

- 方案名称
- 为什么选择该散热路线
- 热源到环境的传热路径
- 预测热点温降
- 预计增重
- 干涉风险
- 材料与制造建议
- 工程风险提示

浮层不是永久侧栏，关闭后恢复完整 3D 舞台。

## 6. 用户流程

### 状态 1：空闲

- 中央显示关节轮廓。
- 底部提示上传文档并描述需求。
- 对话区显示一条简短的 Agent 引导，不显示示例问题列表。
- 用户可以拖入文件或点击上传。

### 状态 2：输入确认

- Agent 用一段短摘要复述识别到的硬件和目标。
- 若关键输入缺失，只追问一个最重要的问题。
- 用户确认后开始生成。

### 状态 3：方案生成

Agent 依次执行：

1. 解析工程文档。
2. 抽取硬件、热源、尺寸、安装边界和目标。
3. 生成基础关节结构参考图。
4. 结合散热方案库优化热增强外壳。
5. 生成多视图提示词和一致性约束。
6. 生成前、左、后、右四向图片；需要时增加顶视图。
7. 检查多视图几何一致性。
8. 将合格图片提交 Hyper3D Rodin。
9. 获取 GLB 并装配到预建关节基座。

生成期间主界面仍只保留最近两条消息；阶段明细进入完整对话与过程抽屉。

### 状态 4：结果呈现

- 中央展示完成的整体关节。
- Agent 只给出三行结论：推荐方案、预计温降、主要风险。
- 页面主操作变为“查看设计”。

### 状态 5：模型检视

- 点击模型触发爆炸分解。
- 点击热增强外壳显示设计说明。
- 再次点击空白处或“合并”恢复整体模型。
- 用户可以导出设计摘要、参考图和 GLB。

## 7. Agent 编排流程

### 7.1 目标架构

Hermes Agent 负责后端编排，前端不直接调用模型服务。

建议的工具边界：

- `parse_engineering_document`
- `build_engineering_brief`
- `select_thermal_strategy`
- `generate_concept_image`
- `build_multiview_prompt`
- `generate_multiview_images`
- `review_multiview_consistency`
- `submit_hyper3d_job`
- `poll_hyper3d_job`
- `assemble_model_artifact`
- `build_design_explanation`

每个工具必须返回结构化结果，不能让下一阶段依赖自然语言猜测字段。

### 7.2 前端与后端通信

使用 BFF 隔离浏览器与外部服务：

- 提交任务：HTTP POST
- 阶段进度：SSE
- 取消任务：HTTP POST
- 查询任务与产物：HTTP GET
- 3D 文件：短期签名 URL

Hyper3D API Key、LLM Key 和图像服务 Key 只能保存在服务端。

### 7.3 阶段恢复

每个阶段独立保存产物：

- 工程摘要
- 热设计方案
- 概念图
- 多视图 Prompt
- 多视图图片
- Hyper3D 任务 ID
- GLB 地址

失败时只重试当前阶段，不重新执行整条链路。

## 8. 3D 模型方案

### 8.1 模型组成

最终场景包含三个独立节点：

1. `joint-base`：预建关节基座。
2. `thermal-interface`：导热垫或连接层。
3. `thermal-shell`：AI 生成的热增强外壳。

爆炸动画只移动独立节点，不对 Hyper3D 返回的单一网格做假分割。

### 8.2 Hyper3D 输入规范

Rodin 输入为同一热增强外壳的多视图图片：

- 顺序固定：前、左、后、右、可选顶视。
- 背景保持纯色或透明。
- 镜头使用近似正交视图。
- 各视图比例、孔位、边缘、散热结构数量保持一致。
- 图中只保留需要生成的热增强外壳，避免把整个关节错误融合为单一网格。
- 输出格式使用 GLB，材质优先 PBR。

### 8.3 装配与对齐

生成模型不能直接被视为尺寸可靠的 CAD。Phase 0 与 Phase 2 采用包围盒归一化：

- 统一关节轴向。
- 将外壳缩放到预设安装包络。
- 使用固定装配锚点定位。
- 超出包络时阻止进入“完成”状态，并提示重新生成。

精确制造尺寸应由后续参数化 CAD 或人工工程复核完成，不由图生 3D 结果直接承担。

### 8.4 交互

- 单击整体：进入爆炸视图。
- 单击部件：选中并高亮。
- 拖动：旋转。
- 滚轮或双指：缩放。
- 双击空白：重置视角。
- 可见按钮提供“爆炸/合并”和“重置”，不能只依赖隐藏手势。
- 开启 `prefers-reduced-motion` 时，爆炸动画改为短淡入淡出。

## 9. 视觉设计系统

### 9.1 风格

风格定义为“精密工业极简”，不是赛博朋克，也不是玻璃拟态仪表盘。

记忆点是：

> 一件悬浮在黑色空间中的关节结构，由 Agent 逐步生成并拆解。

### 9.2 色彩

- 页面背景：近黑石墨色 `#070707`
- 模型舞台背景：`#0B0B0A`
- 主要文字：暖白 `#F2EFE8`
- 次要文字：中性灰 `#8C8A84`
- 品牌强调：热橙 `#FF5A1F`
- 热区高亮：橙红到琥珀色
- 成功状态：低饱和冷绿，仅用于状态，不做大面积装饰

橙色只用于热点、主要动作和 Agent 当前阶段。页面大部分保持黑、灰、暖白。

### 9.3 字体

- 正文与界面：IBM Plex Sans
- 技术参数与阶段编号：IBM Plex Mono
- 字体优先本地托管 WOFF2，避免首屏等待第三方字体。

### 9.4 空间与表面

- 使用 8px 空间基线。
- 主要内容不包卡片。
- 输入框、用户消息和说明浮层可使用单层实体深色表面。
- Agent 消息不使用带边框的大气泡，以文字排版与角色标签区分。
- 最近两条消息位于背景渐隐层上，不添加覆盖整个底部的实色面板。
- 圆角控制在 12～16px。
- 不使用多层发光阴影。
- 背景只允许一个低对比度径向光场，用来衬托模型轮廓。

### 9.5 动效

- 普通反馈：150～220ms。
- 输入框任务态切换：240ms。
- 说明浮层：220ms。
- 模型爆炸：450～600ms，使用可中断的弹性过渡。
- 每个视图最多保留 1～2 个同时运动的重点元素。
- 动效必须表达阶段变化或空间关系，禁止纯装饰循环动画。

## 10. 响应式规则

### 桌面端

- 3D 模型居中，占据主要空间。
- 最近两条消息与输入框位于底部中央。
- 对话层最大宽度约 760px，不能形成左右分栏。
- 说明浮层出现在模型右侧，但不形成永久侧栏。

### 平板端

- 3D 舞台保持全宽。
- 说明浮层改为底部半屏。
- 极简对话区宽度为视口减去 32px。
- 最近两条消息最多各显示三行。

### 移动端

- 3D 舞台优先展示，约占视口高度 55%～65%。
- 最近两条消息与输入框固定在底部安全区内。
- 完整历史与过程使用底部抽屉。
- 软键盘弹出时允许 3D 舞台缩小，但不得完全消失。
- 爆炸控制和重置控制保持至少 44px 点击区域。
- 不允许横向滚动。
- 复杂阴影、抗锯齿级别和模型面数可按性能降级。

## 11. Phase 0 实现范围

Phase 0 的目标不是接通全部模型服务，而是验证产品体验是否成立。

必须实现：

- 单页极简 Agent 布局。
- 中央可旋转、缩放的真实 3D 场景。
- 程序化或本地 GLB 关节基座。
- 独立的热增强外壳节点。
- 爆炸、合并、部件选择和说明浮层。
- 默认只显示最近两条消息的极简对话区。
- 完整对话与过程抽屉。
- 文档上传与设计描述输入框。
- Mock Agent 阶段流。
- 复用现有散热方案和热分析输出作为说明内容。
- 桌面、平板、移动端基础适配。

Phase 0 不实现：

- 真实文档解析。
- 真实 LLM 调用。
- 真实图像生成。
- Hyper3D API 调用。
- Hermes Agent 部署。
- CAD 级尺寸保证。
- 用户账号、云端项目管理和协作。

## 12. 后续阶段

### Phase 1：工程输入与 LLM

- 接入 PDF / DOCX 文档解析。
- 建立结构化工程摘要 Schema。
- 接入 LLM，完成字段抽取、追问和散热策略选择。
- 对输入做尺寸、单位和范围校验。

### Phase 2：图像与 Hyper3D

- 接入图像生成适配器。
- 生成概念图和一致多视图。
- 增加多视图质量门禁。
- 接入 Hyper3D Rodin 提交、轮询和下载。
- 将生成 GLB 装配到预建基座。

图像服务商不影响 Phase 0。Phase 2 开始前通过一次小型对比测试选择能稳定输出一致多视图的模型，而不是提前把 UI 绑定到某一家。

### Phase 3：Hermes Agent

- 将既有串行服务封装为 Hermes 工具。
- 增加任务记忆、阶段恢复和可观测日志。
- 通过 SSE 将工具阶段映射为用户可理解的进度。
- 限制工具循环次数，并为外部调用设置超时、重试和幂等键。

### Phase 4：工程化增强

- 参数化 CAD 或几何约束。
- 结构干涉检测。
- 热仿真或实测校准。
- 版本对比与人工审批。
- 可追溯的设计报告。

## 14. 错误与恢复

- 文档不支持：在输入框附近说明支持格式，不弹全局错误页。
- 文档字段不足：Agent 只追问一个关键问题。
- LLM 失败：保留已解析文档，可重试当前阶段。
- 多视图不一致：不提交 Hyper3D，返回“重新生成视图”。
- Hyper3D 超时：保留图片和任务 ID，允许继续轮询或重新提交。
- GLB 加载失败：展示多视图图片，并提供重新加载。
- WebGL 不可用：降级为可旋转多视图图集，仍可查看设计说明。

## 15. 安全与数据边界

- API Key 仅存储在服务端环境变量中。
- 文件上传限制类型与大小，默认单文件不超过 20MB。
- 文档解析进程需要隔离，拒绝可执行文件和宏。
- 临时文件和签名 URL 设置过期时间。
- 日志不记录原始工程文档、密钥或完整提示词。
- 生成任务带用户或会话归属校验，防止跨任务读取。

## 16. Phase 0 验收标准

### 视觉

- 首屏没有顶部步骤导航、永久侧栏或指标卡网格。
- 中央 3D 模型是首屏最大元素。
- 对话区位于模型下方或底部渐隐层内，不能与模型形成左右工作台。
- 主界面只显示最近两条消息，完整历史默认关闭。
- 首屏可见主要操作不超过三个。
- 页面主要颜色不超过黑、灰、暖白、热橙四组。
- 页面不存在大面积玻璃拟态和装饰性发光。

### 交互

- 用户可在一个页面完成上传、启动、查看进度和检视模型。
- 新消息出现时 3D 舞台不发生明显尺寸跳变。
- 用户可展开和收起完整对话历史，收起后恢复最近两条消息。
- 模型支持旋转、缩放和重置。
- 点击整体可爆炸，点击热增强外壳可打开说明。
- 爆炸状态可可靠恢复。
- 关键操作支持键盘访问和可见焦点。

### 功能

- Mock 流程可从空闲状态稳定推进到完成状态。
- 刷新后能恢复最近一次 Mock 任务状态。
- 现有散热方案数据能出现在部件说明中。
- 错误状态可以从当前阶段重试。

### 质量

- `npm run lint` 通过。
- `npm run test` 通过。
- `npm run build` 通过。
- 375px、768px、1440px 三档视口无横向滚动。
- 开启减少动态效果后仍可理解爆炸和选中状态。

## 17. 最终体验摘要

用户打开 ThermalForge 后，不再面对一套需要逐页填写的工程后台。

他看到的是一个安静的 3D 生成空间：中央悬浮着关节，底部只保留最近两条对话和一个输入框。上传工程资料并描述目标后，Agent 用少量阶段信息解释自己正在做什么，模型在中央逐步成形；更早的上下文只在用户主动展开时出现。生成完成后，用户点击模型让结构分解，再点击热增强外壳查看设计依据、热性能、制造建议和风险。

这就是新 UI 的唯一主线：

> 输入工程资料 → Agent 生成 → 中央 3D 成形 → 爆炸检视 → 查看设计依据
